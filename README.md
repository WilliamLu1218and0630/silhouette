# Silhouette

A minimalist 3D puzzle — rotate the rods so their projection matches the goal silhouette.

**© 2026 William Lu. All rights reserved.** See [LICENSE](LICENSE).

---

## Play

- Web build (GitHub Pages): https://williamlu1218and0630.github.io/silhouette/
- Local dev:
  ```
  npm run dev
  ```
  Then open http://localhost:8765

## Project structure

| Path | Purpose |
| --- | --- |
| `index.html` | Game shell, UI, i18n strings |
| `threeGame.js` | Three.js scene, rod rendering, input |
| `levels.js` | Level data (generated by solver) |
| `geometry.js` | Projection / silhouette geometry |
| `goalPreview.js`, `gen-preview.js` | Goal silhouette rendering |
| `solver/` | Level generator + audit tools (Node ≥ 18) |
| `style.css` | All styles |
| `level-editor.html` | Standalone level editor |

## Roadmap notes (private)

### Going commercial on Google Play / App Store

Pre-launch legal/business checklist when ready to monetize:

**Identity & accounts**
- [ ] Decide entity: individual developer vs. registered business (商號/公司).
      Tax implications differ — solo on App Store works but registering a
      工作室 / 行號 in Taiwan lets you claim development expenses.
- [ ] Apple Developer Program: USD $99/year. Individual or Organization?
      Organization needs D-U-N-S Number (free, ~2 weeks).
- [ ] Google Play Console: one-time USD $25.

**Trademark**
- [ ] Search TIPO (智慧財產局) for existing "Silhouette" marks in classes
      9 (software) and 41 (games as a service).
- [ ] If the bare word is too generic to register, register a **logo lockup**
      or distinctive subtitle (e.g. "Silhouette: Rods").
- [ ] Consider parallel filing in target markets (US: USPTO, JP: JPO) only
      after meaningful traction.

**Contracts & assets**
- [ ] Music: when commissioning, get a written **work-for-hire** assignment
      or perpetual royalty-free commercial license — verbal/email is risky.
      The current `Music: (TBA)` line is a placeholder; replace before launch.
- [ ] SFX: confirm every sound source is either original, CC0, or
      commercially licensed. Keep receipts/license files in `licenses/`.
- [ ] Font: Inter, Nunito, JetBrains Mono are OFL — commercial use is fine,
      but the OFL **prohibits selling the font files standalone**. Bundling
      them inside the app is allowed.

**Store compliance**
- [ ] Privacy policy URL — required by both stores even if you collect
      nothing. Host on GitHub Pages.
- [ ] Data Safety / Privacy nutrition labels: declare zero data collection
      explicitly. If you later add analytics (Firebase, GameAnalytics),
      update labels immediately.
- [ ] Age rating questionnaires (IARC).
- [ ] Tax forms: W-8BEN (Apple/Google US withholding) for individual; or
      營利事業所得稅 setup if registered as a business in Taiwan.

**Monetization model decisions**
- [ ] Paid up-front, free + IAP (level packs / hints / cosmetics), or
      free + non-intrusive ads? Each has different store requirements.
- [ ] If IAP: storefront prices need to be set per region; Apple uses
      Pricing Tiers, Google has more granular control.
- [ ] If ads: pick **one** SDK and disclose it in privacy policy.

**Pre-launch evidence (keep forever)**
- [ ] Tag a release on Git when each major milestone ships — `git tag v1.0.0`.
      Tags + commit timestamps are your strongest authorship evidence.
- [ ] Export and archive the full project on launch day (zip + cloud
      backup with timestamp).
- [ ] Archive a snapshot of the live site on https://web.archive.org/
      as a third-party-witnessed publication date.
